# HTML Studio 项目协作规则

## 技能加载边界

- 默认只使用与软件迭代流程有关的 Superpowers 技能。
- 新增功能、明显行为变更、多步任务和架构调整必须按 Superpowers 流程执行：先设计，再计划、实现、验证和审查。
- 日常问答、代码阅读、状态检查、文案调整、轻量配置以及范围明确的小修，不强制启动完整 Superpowers 流程；直接完成并做与风险相称的验证。
- 非 Superpowers 技能只有在任务本身明确需要其能力时才能加载，例如浏览器用于真实 UI 验证、GitHub 用于 PR 操作、文档技能用于生成对应格式文件。
- 不得因为关键词碰撞加载与本项目任务无关的技能，包括股票分析、技术分析、内容运营、游戏、视频生成等。
- 用户显式指定技能时，以用户要求为准。

## 项目规范

- 开始修改前阅读并遵守 `CODEBUDDY.md` 和 `ITERATION-SOP.md` 中与当前任务相关的规则。
- 完成声明必须有新鲜的测试或真实交互验证证据。
