# 工作区范围与发布上下文设计

**日期：** 2026-07-22
**状态：** 已采用（产品审计第 1、2 轮）

## 问题

HTML Studio 允许多个扫描根。若同时配置父目录与其子目录，当前后端会分别扫描，前端会分别渲染；同一文件因此在树与搜索中重复出现。用户无法快速确认当前搜索范围，也无法快速确认当前文档属于哪个已配置工作区。

Markdown 的公众号操作还存在上下文泄漏：`hidden` 属性不能可靠压过 `.md-wechat-actions` 的显示样式，导致复制、导出动作会在非公众号模式出现。

## 目标

1. 用户可在侧栏选择“全部工作区”或一个已配置扫描根；范围只影响树与搜索的呈现，不改变扫描配置与读写白名单。
2. “全部工作区”不重复展示任何被其他启用根包含的根；较小的子根仍可作为独立工作区选择。
3. 搜索天然遵守当前可见树，因此不会对相同绝对路径显示两次。
4. 顶部显示当前打开文件所属的最长匹配工作区名称；无文件时显示当前工作区范围。
5. 设置页面提示父子扫描根重叠及“全部工作区”下的去重呈现规则，不自动修改用户配置。
6. 公众号复制/导出仅在 `wechat` 视图出现。

## 非目标

- 不删除、禁用、重排或合并用户现有扫描根；
- 不改变 `/api/tree`、`/api/config`、`_resolve_safe()`、快照或保存语义；
- 不新增服务端数据库、账号、网络依赖或 npm 构建流程；
- 不引入完整项目/标签系统。

## 交互设计

### 工作区选择器

侧栏工具栏下方增加紧凑选择器 `#workspace-scope`：

- 默认值：`__all__`，标签为“全部工作区”；
- 每个启用扫描根对应一个 option，label 使用根目录末段；相同末段时显示压缩父路径以避免歧义；
- 选择值存入 `localStorage` 键 `doccenter.workspaceScope.v1`；若保存的路径不再是启用根，自动回退到 `__all__`；
- 切换范围立即重新渲染目录树、清空搜索文本和搜索计数，不发送配置写请求。

### 全部工作区去重

`state.tree` 始终保留后端返回的完整根数组。

- 单一根范围：只渲染该根；
- 全部范围：只渲染不被另一个启用根严格包含的根；
- 若 A 根是 B 根的祖先，A 仍完整显示，因此 B 内的文件仍可访问一次；B 继续出现在选择器中，供用户聚焦。

这个规则解决重复内容，同时不损失可达性和用户对小工作区的快捷配置。

### 当前工作区提示

在顶栏的文件面包屑后新增 `#workspace-context`。

- 打开文件时，用最长前缀匹配确定所属根，例如 `html-doc-center` 优先于 `openclaw-cb`；
- 未打开文件时显示当前选中的范围；
- 文本保留完整根路径到 `title`，可见文本采用 `⌂ 工作区：{label}`；
- 切换范围、重新加载树、语言切换和恢复上次文件时同步刷新。

### 设置提示

扫描目录列表下增加只读 `#scan-overlap-hint`。当一个启用根在另一个启用根内时，显示：

> 已检测到父子工作区。为了避免重复，全部工作区只展示父目录；仍可从工作区选择器进入子目录。

若没有重叠，则不显示该区域。

### 公众号模式

增加 CSS 规则：

```css
.md-wechat-actions[hidden] { display: none; }
```

保留既有 JavaScript 的 `hidden = mode !== 'wechat'` 逻辑；这样原文、分栏、预览模式不会暴露发布动作。

## 架构

全部逻辑位于现有前端：`web/app.js` 从已加载的 `state.tree` 派生展示根，不调用新增 API。

新增纯函数：

```js
isPathInside(path, root) => boolean
getWorkspaceRoots(roots) => Array<{path, name, children, type}>
getDisplayRoots() => Array<Root>
findWorkspaceForPath(absPath) => Root | null
```

路径包含判断须基于已规范化的绝对路径并要求目录边界（`root + '/'`），不能用普通字符串前缀把 `/foo/a` 误判为 `/foo/ab`。

## 验收

### 自动化

- 新增静态契约测试，覆盖工作区选择器、选择持久化键、根去重/最长匹配函数、设置提示、当前工作区提示，以及公众号 `[hidden]` 规则；
- 新增 README 启动命令契约测试或更新已有发布文档测试，使 README 不再宣称不支持的 `--dev` 参数；
- 运行完整 pytest。

### 真实浏览器

1. 用父根和子根同时启用的配置启动；全部范围只出现一个父根，搜索同一路径只有一个结果；
2. 选择子根后，目录树仅显示该子根，搜索只搜索该子根；刷新页面后范围保留；
3. 打开子根文件，顶栏显示子根名称且 title 为完整根路径；
4. 打开设置，看到重叠说明；不点任何开关时配置不变化；
5. 打开 Markdown：原文、分栏、预览模式没有复制/导出按钮；进入公众号模式后按钮出现且预览仍更新；
6. 回归历史抽屉、普通 HTML 编辑、自动快照与关闭三选项。
