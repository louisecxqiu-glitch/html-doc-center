## Goal

让 DocCenter 支持中英文 UI 切换，**首次访问默认英文**（适配 GitHub 海外受众），**localStorage 记忆用户选择**（国内用户切一次后永久中文）。

最终用户能感知的变化：

- 第一次访问 `localhost:9901` → UI 是英文
- 侧栏底部出现 `🌐 EN | 中文` 切换器
- 点击切换 → 全部 UI 文案立刻切换 → 刷新后保持
- 设置面板、对话框、工具栏、Toast、Empty State 全部跟随切换
- CHANGELOG 历史正文保留中文（顶部加英文摘要标注）

## Non-Goals

- ❌ 不翻 CHANGELOG 历史 42 版的中文正文（只翻页面 UI）
- ❌ 不翻被编辑的 HTML 文档内容（用户文档）
- ❌ 不引入 i18next 等 npm 库（保持零 npm 依赖约束）
- ❌ 不改 server.py 的接口字段名（仅前端 UI 翻）
- ❌ 不做 RTL 阿拉伯/希伯来语布局适配（仅 EN/ZH 两语）

## Architecture

### 1. 字典分文件

```
web/
├── i18n.js          # 60-80 行核心：t(), setLang(), applyDOM(), getLangPref()
├── locales/
│   ├── en.js        # window.LOCALE_EN = { ...key-value pairs... }
│   └── zh.js        # window.LOCALE_ZH = { ...key-value pairs... }
```

为什么不放一个文件？两个原因：(a) 后续加日语只新增 ja.js 不动核心；(b) 单文件超 200 行不利人脑维护。

### 2. 核心 API

```javascript
// web/i18n.js（核心 60-80 行）
(function() {
  const STORAGE_KEY = 'doccenter.lang';
  const SUPPORTED = ['en', 'zh'];
  const DEFAULT = 'en';  // GitHub 首印象用英文

  let currentLang = getLangPref();
  const dicts = { en: window.LOCALE_EN, zh: window.LOCALE_ZH };

  function getLangPref() {
    const saved = localStorage.getItem(STORAGE_KEY);
    if (saved && SUPPORTED.includes(saved)) return saved;
    return DEFAULT;
  }

  function setLang(lang) {
    if (!SUPPORTED.includes(lang)) return;
    currentLang = lang;
    localStorage.setItem(STORAGE_KEY, lang);
    document.documentElement.lang = lang;
    applyDOM();
    window.dispatchEvent(new CustomEvent('langchange', { detail: { lang } }));
  }

  function t(key, vars) {
    let s = dicts[currentLang]?.[key] || dicts[DEFAULT]?.[key] || key;
    if (vars) {
      for (const [k, v] of Object.entries(vars)) {
        s = s.replace(new RegExp(`{${k}}`, 'g'), v);
      }
    }
    return s;
  }

  function applyDOM() {
    document.querySelectorAll('[data-i18n]').forEach(el => {
      const key = el.dataset.i18n;
      el.textContent = t(key);
    });
    document.querySelectorAll('[data-i18n-title]').forEach(el => {
      el.title = t(el.dataset.i18nTitle);
    });
    document.querySelectorAll('[data-i18n-placeholder]').forEach(el => {
      el.placeholder = t(el.dataset.i18nPlaceholder);
    });
    document.querySelectorAll('[data-i18n-html]').forEach(el => {
      el.innerHTML = t(el.dataset.i18nHtml);
    });
  }

  window.i18n = { t, setLang, getLang: () => currentLang, applyDOM };
  document.documentElement.lang = currentLang;
  document.addEventListener('DOMContentLoaded', applyDOM);
})();
```

### 3. HTML 改造模式

```html
<!-- 之前 -->
<button title="刷新（R）">🔄</button>

<!-- 之后 -->
<button data-i18n-title="btn.refresh">🔄</button>
```

```html
<!-- 之前 -->
<span class="label">目录</span>

<!-- 之后 -->
<span class="label" data-i18n="sidebar.tab.tree">Files</span>
<!-- 内文留 fallback，便于无 JS 也能看 -->
```

### 4. JS 动态文案改造模式

```javascript
// 之前
showToast('已保存');

// 之后
showToast(window.i18n.t('toast.saved'));

// 带参数
showToast(window.i18n.t('toast.snapshot.created', { count: 5 }));
```

### 5. 切换器 UI 位置

侧栏底部，状态栏下方加一行：

```html
<div class="lang-switcher" id="lang-switcher">
  <button data-lang="en" class="lang-btn">EN</button>
  <span class="lang-divider">|</span>
  <button data-lang="zh" class="lang-btn">中文</button>
</div>
```

样式：12px 字号 / 与状态栏底色融合 / 当前语言加粗 + 主题色。

## Tech Stack

| 维度 | 选型 | 理由 |
|---|---|---|
| 框架 | 手写 60-80 行 vanilla JS | 保持零 npm 依赖约束 |
| 字典格式 | flat key (e.g. `dlg.save.overwrite`) | 比嵌套对象 grep / 重命名更易 |
| 持久化 | localStorage | 客户端足够，无需 server 接口 |
| HTML 标记 | `data-i18n` / `data-i18n-title` / `data-i18n-placeholder` / `data-i18n-html` | 标准 HTML5 data-attr，浏览器兼容性满分 |
| Fallback 链 | 当前语言 → 英文 → key 字符串本身 | 兜底保证不出现 `undefined` |

## 风险与回退预案

| 风险 | 缓解 |
|---|---|
| 翻译质量不达标（机翻味浓） | 先翻 30 条最高频文案，让 Louis review，pass 后批量翻剩余 |
| 字典 key 命名分歧 | 用扁平命名约定 `区域.子模块.元素` 三段，所有 key 进字典前 grep 重复 |
| 切换后部分动态 DOM 未更新 | dispatchEvent('langchange')，相关模块监听重渲染 |
| 性能下降（每次 t() 都查 dict） | dict 是 plain object lookup，O(1)，可忽略 |
| 后续加日语成本 | 当前架构只新增 ja.js 即可，0 改动核心 |

## 回滚方案

如果 v1.12.0 上线后发现严重 bug：
1. `git revert <v1.12.0 commit>` 回到 v1.11.11
2. 用户体验：localStorage 中残留 `doccenter.lang` key 不影响 v1.11.11 运行（v1.11.11 没读这个 key）
3. 若需彻底清理：v1.12.1 hotfix 加一行 `localStorage.removeItem('doccenter.lang')`
