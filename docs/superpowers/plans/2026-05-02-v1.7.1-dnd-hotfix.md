# v1.7.1 — 拖拽 Hotfix + 缓存头加固

## Goal

修 v1.7.0 拖拽的 3 个体验 bug + 一个系统性缓存问题：

- **A** 移动成功后目标目录不自动展开，用户不知道新文件去哪了
- **B** Dirty 文件拦截 toast 出现在右下角不显眼（用户在左侧拖拽看不见）
- **C** Dirty 文件被拖时节点变灰状态残留，保存后仍不恢复
- **D**（基建）静态资源缓存策略不够狠，改代码后浏览器老看到旧版本

## File Structure

| 文件 | 改动 |
|---|---|
| `web/app.js` | Modify — dragstart 增强 dirty 保护；handleDrop 成功后 expandToPath + flash；toast 顶部中央（用户选 c） |
| `web/style.css` | Modify — .toast-container 顶部中央；.flash-highlight 复用；新增 .dnd-dragging 兜底清除 |
| `server.py` | Modify — `/`、`/static/*`、`/saver-runtime.js` 响应加 Cache-Control: no-store, no-cache, must-revalidate |
| `web/index.html` | Modify — 版本号 bump v1.7.1 |
| `CHANGELOG.md` + `CHANGELOG.html` | Modify — v1.7.1 卡片 |
