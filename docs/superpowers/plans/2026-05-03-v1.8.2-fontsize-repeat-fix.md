# v1.8.2 — 字号第二次不生效 Hotfix

> For agentic workers: REQUIRED SUB-SKILL: Use superpowers:systematic-debugging + verification-before-completion.

## 问题
选中文字 → 字号选 14（生效）→ 再选 18 → **不生效**。字色同样有概率复现。

## 根因（三段式）
- **问题**：`applyInlineStyleToSelection` 成功执行后，DOM 被 `extractContents` + `insertNode` 重组，旧 range 指向的节点已失效。
- **根因**：`_savedRange` 只在 `mousedown` 时刷新，但第二次点击 `<select>` 前，浏览器的真实 selection 在 select 拿焦时已经 collapse → `saveSelection()` 的 `!isCollapsed` 守卫跳过更新 → `_savedRange` 仍是上次"已经被破坏"的旧 range → restore 后 `sel.isCollapsed===true` → `applyInlineStyleToSelection` 直接 return false。
- **解法**：改用 `document.addEventListener("selectionchange")` 持续追踪最新非空选区；`applyInlineStyleToSelection` 成功后把新建的 range 同步回 `_savedRange`，形成闭环。
