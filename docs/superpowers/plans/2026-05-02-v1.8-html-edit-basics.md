# v1.8.0 — HTML 编辑增强（第一批）：图片替换 + 字号/颜色/加粗

## Goal

解决用户核心痛点——"AI 生成的 HTML 我不满意但改不动"。本批先覆盖两类最高频场景：

1. **图片编辑**：选中图片 → 替换 / 删除 / 粘贴新图（剪贴板或本地文件）
2. **文字属性**：字号（S/M/L 档 + 自定义 px）、字色（预设色板 + 自定义）、加粗（已有，升级 UI）

**图片存储策略**（默认）：**base64 内嵌**，保持 DocCenter 单文件自包含哲学。未来可选独立文件模式（v1.9+）。

**修改粒度**：在选中元素上加 inline style，不动原 CSS 类。便于还原和迁移。

## Architecture

本批完全在前端 `saver-runtime.js` 内实现，后端零改动（沿用现有 `/api/snapshot` 和 `/api/save`）。

```
selectionManager（新增）
  ├─ 监听点击/selectionchange
  ├─ 维护 state.selectedImg 和 state.selectedTextRange
  └─ 触发对应浮窗显示/隐藏

imagePopover（选中图片时出现在图片上方）
  ├─ 替换（打开文件选择器 or 粘贴剪贴板）
  ├─ 删除（element.remove() + markDirty）
  └─ 关闭

textFormatBar（选中文字时增强现有顶栏）
  ├─ 字号下拉（12/14/16/18/24/32/48）
  ├─ 字色色板（8 预设 + 自定义）
  └─ 已有 B/I/U 改为高亮激活态

imagePasteHandler（全局监听）
  ├─ ⌘V 粘贴图片 → 自动插入到光标位置
  └─ 拖入图片文件 → 同上
```

## Tech Stack

- `document.execCommand('fontSize' | 'foreColor')` 实现文字属性（老 API 但够用，未来可迁移 Selection API）
- `FileReader.readAsDataURL` 转 base64
- `ClipboardEvent.clipboardData.items` 读剪贴板图片
- 原生浮窗（`position: absolute` + getBoundingClientRect 定位）

## 回退预案

`git revert v1.8.0-commit`，saver-runtime.js 回到 v1.7.0 状态，用户编辑的图片/文字在 overwrite 之前都有 `_auto-save/` 快照可恢复。

## 已知限制（v1.8.0 不处理，留给 v1.9）

- 不支持图片拖拽改位置（仅能删除后粘贴到新位置）
- 不支持图片缩放（修改 width/height），未来由 v1.9 局部布局覆盖
- 不支持独立文件图片模式（默认 base64）
- 不支持段落/区块的布局调整（padding/margin/layout），留给 v1.9 D-α 方案
- 不支持 undo/redo 跨操作栈（仅依赖浏览器原生 contenteditable undo）
