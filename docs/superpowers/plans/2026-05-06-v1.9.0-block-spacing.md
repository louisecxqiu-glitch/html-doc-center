# v1.9.0 · 块级间距调整（Block Spacing Editor）

> For agentic workers: REQUIRED SUB-SKILL: Use superpowers:test-driven-development + verification-before-completion.

## Why（痛点）

用户在 DocCenter 改 AI 生成的 HTML 报告时遇到：

> "想把这个页面的下半部分和上半部分挪开一些，距离大一些"

当前能力覆盖率：**布局类 0%**（6 项都没）。用户被迫：
- 回上游 AI 重生成（违反 G3「零 AI 依赖」目标）
- 或放弃调整（体验差）

这是路线图 `2026-05-06-roadmap-and-gap.md` 里识别的"第三关：调版"的第一个具体落地点。

## What（要做什么）

给 DocCenter 加**第三种编辑上下文**：

| 上下文 | 触发 | 现有？ |
|---|---|---|
| 文字内选区 | 选中文字 | ✅ v1.8 |
| 点击图片 | 点 `<img>` | ✅ v1.8 |
| **点击块** | Alt/⌥ + 点任意块级元素 | ❌ 本版新增 |

选中块后浮窗提供：
- 上边距 / 下边距 滑杆（0-120px，步进 4px）
- 快捷按钮：「紧凑」「默认」「宽松」「极宽」
- 方向键微调：`↑↓` 改上边距 ±4px，`←→` 改下边距 ±4px
- 清除 inline margin 按钮

## How（技术方案）

### 核心设计

沿用 `ImageEditor` 的 IIFE 模块架构，**照搬骨架**，只换交互对象：

```js
const BlockSpacingEditor = (function () {
  let selectedBlock = null;
  let popover = null;

  function init() {
    document.addEventListener("click", onClick, true);
    document.addEventListener("keydown", onKeyDown, true);
  }
  // ... 类比 ImageEditor 的十几个方法
})();
```

### 触发方式（关键 UX 决策）

**不用纯 click** —— 否则用户编辑文字时会误触。改用 **Alt/⌥ + click** 进入"块选择模式"：

```
普通 click：编辑文字（保持现有行为）
⌥ + click：选中最近的块级祖先 → 浮窗
Esc / 点空白：退出块模式
```

备选方案（plan 内记录，待决策）：
- A. 工具栏加一个 "📐 布局" 模式开关（toggle）
- B. 三击 triple-click 进入块模式
- C. 右键菜单「调整间距」

默认实施 **⌥ + click**（最低摩擦，键盘熟练用户友好）。同时留工具栏小按钮 `📐` 作为"不会快捷键"的兜底。

### 块级祖先查找

```js
function findBlockAncestor(el) {
  const BLOCK_TAGS = new Set([
    "DIV","SECTION","ARTICLE","ASIDE","HEADER","FOOTER","NAV","MAIN",
    "P","H1","H2","H3","H4","H5","H6",
    "UL","OL","LI","TABLE","FIGURE","BLOCKQUOTE","PRE","HR"
  ]);
  while (el && el !== document.body) {
    if (BLOCK_TAGS.has(el.tagName) && el.offsetHeight > 0) return el;
    el = el.parentElement;
  }
  return null;
}
```

### 间距应用方式

**只用 inline style**，不碰原 CSS 类：

```js
block.style.marginTop = `${value}px`;
block.style.marginBottom = `${value}px`;
```

清除时删除 inline `margin-top/bottom`，让原 CSS 重新生效。

### 浮窗 UI

锚定在选中块的右上角（类比 ImageEditor 的图片浮窗）：

```
┌─ 📐 块间距 ─────────────────────┐
│                                │
│  上边距  [====●========]  24px │
│         0          120         │
│                                │
│  下边距  [==========●==]  48px │
│         0          120         │
│                                │
│  [紧凑] [默认] [宽松] [极宽]     │
│                                │
│  ⎌ 清除  · ✕ 关闭               │
└────────────────────────────────┘
```

预设：
- 紧凑 = 上 8, 下 8
- 默认 = 上 16, 下 16
- 宽松 = 上 40, 下 40
- 极宽 = 上 80, 下 80

## 风险 & 已知限制

- **选中块误判**：`findBlockAncestor` 在嵌套 flex/grid 布局下可能选到父容器而非目标。缓解：浮窗标题显示选中的 selector（`div.section > .card`），让用户看得见选了啥，选错可 Esc
- **margin 合并（collapse）**：相邻块的 margin 会合并（CSS 规则），滑杆显示 40px 实际可能只看到 40px 而不是 80px。缓解：在浮窗加"⚠️ 检测到 margin collapse"小字提示（v1 可先不做，v1.9.1 补）
- **块有原生 margin**：用户以为"清除 = 回到 0"，实际是"回到原 CSS 值"。缓解：清除按钮显示为「↺ 恢复原始间距」而非「清零」

## 不做（明确排除）

- ❌ **左右边距** / **padding** / **缩进**：scope 膨胀，v1.10 再说
- ❌ **拖拽改间距**（类似 Webflow）：复杂度高，滑杆够用
- ❌ **块移动位置/排序**：大动作，单独开 v1.9.6 plan
- ❌ **批量选择多块调间距**：v2.0 "节奏修复器"的事
