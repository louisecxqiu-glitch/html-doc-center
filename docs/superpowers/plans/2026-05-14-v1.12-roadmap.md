## v1.12 候选清单（按优先级 P1 高 → P3 低）

### 🟢 P1 · 大概率会做

#### v1.12.0 (候选): 表格编辑（P12）
- 选中表格时浮出工具栏：插行/列、删行/列、合并/拆分单元格
- 工程量：大（鼠标交互复杂、合并状态管理）
- 真实痛点：做技术 deck 经常需要调整表格结构

#### v1.12.1 (候选): 内容搜索（不只是文件名）
- /api/search-content 全文搜索接口
- 前端在搜索框输入 ":content xxx" 触发内容模式
- 工程量：中（要做索引或边读边搜）
- 价值：找"我之前写过的 xxx 那段话在哪个文档"

#### v1.12.2 (候选): 跨文件 diff
- 选中两个文件 → 对比内容 diff（不是同一文件的两个快照）
- 用例：对比"v1 封面 vs v2 封面"两个独立 HTML
- 工程量：中

### 🟡 P2 · 视情况做

#### v1.12.3 (候选): 智能撤销栈（P7 完整版）
- 不是单纯快照恢复，而是把"DocCenter 内每次操作"做成可撤销 stack
- 跨多个 contenteditable 操作的统一 undo（比浏览器原生 undo 强）
- 工程量：很大（要 hook 所有 mutation 写 undo entry）
- 风险：v1.2.4 脏状态护栏可能受影响

#### v1.12.4 (候选): 文件历史进度条
- 在 iframe 顶部（saver 工具栏附近）画一条"写作时间线"
- 展示该文件从创建到现在的 N 个快照分布
- 用户可以拖滑块到任意时点 → 预览
- 工程量：中（saver-runtime 内新增 UI 模块）

#### v1.12.5 (候选): 评论/讨论模式
- 不是 v1.8 的批注（那是个人备注），而是"模拟多人评论"
- 比如标注"这段我觉得可以删，路易"，用@签名
- 用例：自己写完后假装别人 review，强迫切换视角
- 工程量：中

### 🔵 P3 · 长期备选

- 主题色自定义（用户传入金色之外的品牌色）
- 拖拽多选 + 批量操作（v1.10 之前已盘点过）
- 全文搜索结果独立视图（v1.10 阶段已否决）
- WebDAV 同步（违反"本地优先"理念，慎重）

### 🔴 不做（明确否决）

- 云同步：本项目核心是"本地、单机、零依赖"
- 多账户：本项目是个人工作台
- 实时协作：和定位冲突
