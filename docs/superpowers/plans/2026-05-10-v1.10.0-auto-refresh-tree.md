## Goal

让 DocCenter 侧边栏目录树**定期自动刷新**，新增/删除/重命名 HTML 文件无需手动按"刷新"按钮即可在 N 秒内自动出现/消失。同时**不打断用户当前的编辑/批注/选中态**。

## Why（场景）

路易在创作时常出现：
1. 在终端用 AI 生成新 HTML 到 `outputs/` 下
2. 切到 DocCenter 想立即点开新文件
3. **侧边栏看不到新文件**，必须手动点 🔄 才能看到
4. 每次都要切换工具，打断创作流

类似场景：另一篇 HTML 被删除/移动后，侧边栏仍显示死链。

## Architecture（核心思路）

**前端轮询 + 后端 force refresh**（不引入 watchdog/SSE/WebSocket）：

```
[前端 app.js]
  ↓ 每 10 秒
  fetch(/api/tree?force=1&sort=...)
  ↓
[后端 server.py]
  → get_tree(force=True) → _scan_tree() 重扫磁盘
  ↓
[diff & 渲染]
  → 只更新变化的节点（增/删），保持当前展开态、选中态、收藏星标
```

### 关键设计决策

| 决策 | 选择 | 理由 |
|---|---|---|
| 轮询 vs 推送 | **前端轮询** | 改动最小，零新依赖，符合项目轻量原则 |
| 周期 | **默认 10 秒**（可配置 5/10/30/60s/关闭） | 平衡响应感和性能 |
| 页面可见时 | 轮询 | Chrome `document.visibilityState === 'visible'` 守护 |
| 页面隐藏时 | **暂停**轮询 | 节省电量，切回时立即触发一次 |
| 当前在编辑 | 仍轮询，**只更新树 DOM** | 不动 iframe、不动选中态、不动滚动位置 |
| 用户操作树时 | 不打断 | 用户最近 3 秒有树交互（hover/click），跳过本次刷新 |
| 失败处理 | 静默重试 | 不弹 toast，console.warn 即可 |
| 配置入口 | 设置面板新增"自动刷新"项 | 与现有 `snapshot_debounce_ms` 风格一致 |

### Diff 算法

不重新渲染整棵树（会破坏展开态/收藏态），而是：
1. 拿到新 tree → 递归生成 `flatPathSet`（所有节点的相对路径集合）
2. 对比当前 DOM 中已渲染的 `data-path`：
   - **新增**的节点：插入到对应父级，淡入动画
   - **删除**的节点：从 DOM 移除，淡出动画
   - **未变**的节点：完全不动（保持展开/选中/星标态）
3. 不处理重命名（视为删除+新增，简单可靠）

## Tech Stack

无新依赖。完全基于现有：
- 后端 `aiohttp` + `_scan_tree()`（已支持 `force` 参数）
- 前端原生 JS（`fetch` + `setInterval` + `document.visibilitychange`）

### Task 3：Diff 渲染（不破坏 UI 状态）
**Why**：直接 `renderTree()` 会清空现有 DOM，导致用户展开的目录被收起、选中的文件失焦。需要做精确 diff。

**Files**：`web/app.js`

**Steps**：
- [x] 3.1 在 `loadTree()` 增加 `silent: bool` 参数；`silent=true` 时调用 `diffTreeAndPatch()` 而非 `renderTree()`
- [x] 3.2 实现 `flatTree(roots) → Map<path, {node, parent}>`，递归扁平化
- [x] 3.3 实现 `diffTreeAndPatch(newRoots)`：
  - 拿到 `oldFlat = flatTree(state.tree)` 和 `newFlat = flatTree(newRoots)`
  - 计算 `added = newFlat.keys - oldFlat.keys`
  - 计算 `removed = oldFlat.keys - newFlat.keys`
  - 对 added：找到父节点 DOM，调 `renderEntry(node)` 插入，加 class `tree-node-fade-in`
  - 对 removed：`querySelector([data-path="..."])` → `.classList.add("tree-node-fade-out")` → 200ms 后 remove
  - 不变的节点完全不动
- [x] 3.4 `state.tree` 更新为 `newRoots`，刷新收藏星标

**验收**：
- 终端 `touch outputs/test-new.html` → 10 秒内侧边栏出现 test-new.html，已展开的目录保持展开
- `rm outputs/test-new.html` → 10 秒内消失
- 当前选中的文件 + iframe 内容完全不动

## Risk & Mitigation

| 风险 | 缓解 |
|---|---|
| 大目录每 10 秒扫一次性能差 | 后端已有 entry-only 优化；用户可调 30/60s 或关闭 |
| 用户正在拖拽节点时 diff 闪烁 | `lastUserActivityTs` 守护，3 秒内有交互不刷新 |
| iframe 内容被刷新冲掉 | 永远不动 iframe，只动 `#tree` DOM |
| 浏览器 tab 后台时电量消耗 | `visibilitychange` 自动暂停 |
| 网络抖动一次失败 | 静默 console.warn，下次重试 |

## Success Metrics

- 路易在终端 `cp` 一个新 HTML 到扫描目录后，**≤ 10 秒内**在侧边栏看到，无需手动点刷新
- 用户当前的展开 / 选中 / 编辑态 0 干扰
- 后台标签页时不发请求（验证：DevTools Network 静默）

## Out of Scope（v1.11+ 再做）

- watchdog 文件系统监听 + SSE 实时推送（性能更优但复杂）
- 重命名识别（目前视为删除+新增）
- 远程目录支持（本地文件足够）
