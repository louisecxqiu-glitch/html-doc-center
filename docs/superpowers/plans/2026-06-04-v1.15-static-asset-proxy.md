## Goal

让 DocCenter 打开的 HTML 文件中，引用同目录或子目录下的本地资源（图片、CSS、JS、字体、视频等）能正常加载——与直接在浏览器打开 `file://` 时的表现一致。

**用户能感知的变化**：

- 之前：HTML 引用 `assets/persona-exec.png` → DocCenter 里 404，图片空白
- 现在：HTML 引用 `assets/persona-exec.png` → DocCenter 里正常显示

**触发场景**：任何 HTML 文件通过相对路径引用了同目录/子目录下的静态资源。

## Non-Goals

- ❌ 不支持引用 scan_roots 范围外的资源（安全红线）
- ❌ 不支持 `../` 向上穿越出 scan_root（安全红线）
- ❌ 不代理绝对路径引用（如 `file:///...` 或 `http://...`）——这些走浏览器自身处理
- ❌ 不改变 .md 文件的处理逻辑（md 壳子自包含，暂不涉及）
- ❌ 不做 MIME type 猜测的完美覆盖——用 Python 标准库 `mimetypes` 即可

## Architecture

### 核心设计

```
用户打开 HTML → inject_saver() 注入 <base href="/api/asset/{encoded_dir}/">
  ↓
HTML 中 <img src="assets/foo.png">
  ↓
浏览器解析为 GET /api/asset/{encoded_dir}/assets/foo.png
  ↓
handle_asset() 拼合 base_dir + relative_path → 绝对路径
  ↓
_resolve_safe() 校验在 scan_roots 内
  ↓
返回文件内容 + 正确 Content-Type
```

### 关键设计决策

1. **用 `<base href>` 一行注入解决所有相对路径**——无需逐个重写 `<img src>`、`background: url()`、`<link href>` 等
2. **base 路径用 URL-safe Base64 编码目录路径**——避免目录路径中的特殊字符（中文、空格）干扰 URL 解析
3. **安全校验复用 `_resolve_safe()`**——确保代理的资源必须在 scan_roots 白名单内
4. **`<base>` 注入位置在 `<head>` 最前面**——确保后续所有相对引用都生效
5. **对 saver-runtime.js 的 `src="/saver-runtime.js"` 无影响**——因为它是绝对路径（以 `/` 开头），不受 `<base>` 影响

### 对现有功能的影响

- `/saver-runtime.js`、`/static/*`、`/api/*` 等绝对路径引用不受 `<base>` 影响（以 `/` 开头绕过 base）
- `inject_saver()` 注入的 script 标签已经是绝对路径 `src="/saver-runtime.js"` → 无影响
- HTML 中已有 `<base>` 标签的情况：检测到已有 `<base href>` 时跳过注入（尊重原作者设置）

## Tech Stack

| 维度 | 选型 | 理由 |
|---|---|---|
| 编码方式 | Python `base64.urlsafe_b64encode` | 标准库，URL 安全 |
| MIME 推断 | Python `mimetypes.guess_type` | 标准库，覆盖主流格式 |
| 路由匹配 | aiohttp `/{tail:.*}` 通配 | 支持任意深度子路径 |

## Tasks

### Task 1：新增 `/api/asset/{encoded_dir}/{relative_path:.*}` 路由

**Why**：这是核心——代理本地静态文件的入口。

**Files**：`server.py`

- [ ] **Step 1.1**：在 `server.py` 顶部 import `base64` 和 `mimetypes`
- [ ] **Step 1.2**：新增 `handle_asset(request)` async 函数：
  ```python
  async def handle_asset(request):
      """
      GET /api/asset/{encoded_dir}/{path:.*}
      代理 HTML 同目录/子目录下的静态资源。
      encoded_dir = urlsafe_b64encode(目录绝对路径.encode()).decode()
      """
      encoded_dir = request.match_info["encoded_dir"]
      rel_path = request.match_info.get("path", "")
      
      # 解码 base 目录
      try:
          base_dir = base64.urlsafe_b64decode(encoded_dir.encode()).decode("utf-8")
      except Exception:
          return web.Response(status=400, text="Invalid base directory encoding")
      
      # 拼合绝对路径
      full_path = Path(base_dir) / rel_path
      full_path = full_path.resolve()
      
      # 安全校验：必须在 scan_roots 内
      cfg = request.app["config"]
      safe = _resolve_safe(str(full_path), cfg.get("scan_roots", []))
      if not safe or not safe.is_file():
          return web.Response(status=403, text="Access denied or file not found")
      
      # 不代理 .html / .md（这些走 /api/file）
      if safe.suffix.lower() in (".html", ".md"):
          return web.Response(status=403, text="Use /api/file for HTML/MD")
      
      # 推断 MIME type
      mime, _ = mimetypes.guess_type(str(safe))
      if not mime:
          mime = "application/octet-stream"
      
      # 返回文件
      return web.FileResponse(safe, headers={"Content-Type": mime})
  ```
- [ ] **Step 1.3**：在路由注册区域新增：
  ```python
  app.router.add_get("/api/asset/{encoded_dir}/{path:.*}", handle_asset)
  ```
- [ ] **Step 1.4**：验证路由注册语法正确（启动服务不报错）

      """生成 <base href> 指向文件所在目录的资源代理 URL"""
      dir_path = str(Path(file_path).parent)
      encoded = base64.urlsafe_b64encode(dir_path.encode("utf-8")).decode("utf-8")

## 风险与回退

| 风险 | 缓解 |
|---|---|
| `<base href>` 影响 HTML 中的锚点链接（`<a href="#section">`） | 锚点以 `#` 开头，不受 base 影响 |
| `<base href>` 影响 HTML 中的绝对路径引用 | 绝对路径（`/` 或 `http://` 开头）不受 base 影响 |
| base64 编码后路径过长导致 URL 超限 | 实测 scan_root 路径一般 < 200 字符，编码后 < 300，远低于浏览器 URL 限制 |
| HTML 中有 JavaScript 动态加载资源（fetch 相对路径） | `<base href>` 对 fetch/XHR 的相对 URL 同样生效 |

**回滚方案**：`git revert` 即可。`<base href>` 不存储到磁盘（只在运行时注入），不影响源文件。
