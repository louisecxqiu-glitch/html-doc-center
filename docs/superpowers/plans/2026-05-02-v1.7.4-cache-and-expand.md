# v1.7.4 — ETag 根治 + 展开策略修正

## Goal

两个 bug 打包修：

1. **Z · 浏览器看老版本 bug 彻底根治**（v1.7.1 只加了 Cache-Control 头，但 aiohttp FileResponse 默认带 ETag + Last-Modified，浏览器发 If-None-Match 拿到 304 就用缓存——等于缓存头白加）
2. **A · 拖拽移动后"所有子目录展开"**（v1.6.1 遗漏：renderNode depth=1 没默认 collapsed，当 expandAndFlashPath 展开扫描根后，所有一级子目录默认全开）

## Architecture

### Fix Z — ETag 根治
把 `handle_index / handle_static / handle_saver_js / handle_changelog / handle_changelog_raw` 的 `FileResponse` 改为**读文件 + `web.Response(body=..., content_type=...)`**，不走 FileResponse 避免 ETag/Last-Modified 自动注入。

或更简单：拿到 FileResponse 后 `del resp.headers["ETag"]` + `del resp.headers["Last-Modified"]`。

### Fix A — renderNode depth=1 默认折叠
有 currentFile 时，`depth === 1` 也加 collapsed，由 scrollToActiveFile 单独展开当前文件的父链。

### Fix B — 合并 scrollToActiveFile 和 expandAndFlashPath
如果被移动的是 currentFile，两个函数作用重叠，去重即可：
- 被移动 == currentFile：只调 expandAndFlashPath（它会展开父链并带高亮）
- 被移动 != currentFile：先 scrollToActiveFile 恢复当前文件展开，再 expandAndFlashPath 展开新位置
