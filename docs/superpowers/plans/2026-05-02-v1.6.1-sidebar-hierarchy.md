# v1.6.1 — 侧边栏信息层级重构（Nielsen #6 + 格式塔）

## Goal

解决 v1.6.0 发布后路易用"路易乔布斯视角"诊断出的 5 个信息层级问题：

- P1 多个扫描根横排，当前根埋在中间
- P2 目录和文件混排视觉无区分
- P3 每行重复的 `〔根〕` 文字冗余
- P4 长文件名 tooltip 不明显
- P5 "当前打开文件所在根"无视觉暗示

## Architecture

纯前端改动：CSS 层级 + renderNode 逻辑。**不动后端、不动数据结构、不动 API**。

## Tech Stack

原生 JS + CSS。

## File Structure

| 文件 | 改动 | 说明 |
|---|---|---|
| `web/app.js` | Modify | renderNode 新增"active 根识别 + 强制展开其他根折叠"逻辑；createDirLabel 加 (N) 文件计数；去 〔根〕 文字 |
| `web/style.css` | Modify | .tree-root.is-active / .tree-node-label.in-active-chain / .tree-dir-count / 字号分层 |
| `web/index.html` | Modify | 版本号 bump 1.6.1 |
| `CHANGELOG.md` + `CHANGELOG.html` | Modify | v1.6.1 卡片 |

## Tasks

### Task 1 — 识别 active 根 + 展开策略（Q1=c）

**Files**: `web/app.js`

**Steps**:
- [ ] 1.1: 新增 `isActiveRoot(rootNode)` 判定：当前 `state.currentFile?.absPath` 以该根 `abs_path + '/'` 开头（或相等）
- [ ] 1.2: renderTree 第一次渲染时：active 根展开（不加 collapsed），其他根默认折叠（加 collapsed）
- [ ] 1.3: active 根容器加 `.is-active` 类
- [ ] 1.4: 切换当前文件时只更新 `.is-active` 类和折叠态，不重新 render 整棵树（避免 state 丢失）——通过 `refreshActiveRootHighlight()` 函数
