# DocCenter v1.11 编辑体验与导航精炼 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 复盘 v1.10 期间用户实际使用中暴露的四大顽疾——CHANGELOG 信息密度爆炸、富文本编辑器能力残缺（字色/对齐/链接）、侧栏三区挤兑、图片与元素无法可视化操纵——通过 5 个子版本分阶段解决，每个独立发布、独立回滚。

**Background / 为什么做：**
- 用户 2026-05-13 凌晨复盘提出 4 个痛点 + 截图揭露 2 个新痛点
- 经 brainstorming 后确认优先级：导航 → 编辑 → 高级可视化
- v1.10.11 刚立了"👤 用户故事"铁律，v1.11 每个子版本都要遵守

**Architecture:**
- v1.11.0 / .1：纯前端 CSS + saver-runtime.js 改动，零后端变动
- v1.11.2：侧栏重构，分区 Tab 模式（收藏 / 最近 / 目录 三选一），排序折进搜索栏
- v1.11.3：图片 Notion 式操纵 —— 选中时出现 resize handle，段内上下移
- v1.11.4：元素位置微调 —— 用 `transform: translate(X, Y)` 方案，避免 margin/padding 破坏布局

**Tech Stack:** 纯前端改动，Python 3 + aiohttp 后端不动；每个子版本手工 curl 验证 + 浏览器视觉校验。

## Task 2 · v1.11.1 · 富文本工具栏升级（字色完整方案 + 对齐 + 链接）

**Why:** saver-runtime 现在的字色只有几个固定选项（无白色），没有对齐按钮，没有链接功能。用户改深色封面时字色无白完全不可用。

**Scope:** 几乎全在 `saver-runtime.js` 内改动；可能顺手加一些 CSS 样式。

**Files:** `saver-runtime.js`

### Steps

- [ ] Step 2.1: 读当前 saver-runtime.js 的工具栏 DOM 构建段，了解现有字色实现
- [ ] Step 2.2: 设计完整色板 UI
  - 12 色预设（黑/白/灰/红/橙/黄/绿/青/蓝/紫/粉/金）分两行排列
  - 底部一行"最近使用颜色"（localStorage 存，上限 6 个）
  - 底部"更多颜色…"触发原生 `<input type="color">` 作为吸管替代
- [ ] Step 2.3: 实现字色应用逻辑
  - 优先用 `document.execCommand('styleWithCSS', ...)` + `foreColor`
  - 保留 selectionRange（复用 v1.8.1 已有的 _savedRange 机制）
- [ ] Step 2.4: 新增对齐按钮（4 个：左/中/右/两端）
  - 用 `document.execCommand('justifyLeft' / 'justifyCenter' / 'justifyRight' / 'justifyFull')`
  - 活跃态反馈（当前光标所在段落的对齐方式反向高亮）
- [ ] Step 2.5: 新增链接按钮
  - 选中文字后点击弹 prompt 输入 URL
  - 空 URL 取消链接（`unlink`），非空 URL 加链接（`createLink`）
  - URL 自动加 `https://` 前缀（如果缺失）
- [ ] Step 2.6: 最近使用颜色逻辑：每次选颜色 → unshift 到数组 → 去重 → 截断 6 条 → 写 localStorage
- [ ] Step 2.7: 调色板触发方式：点击字色按钮弹出小浮层，点外部/再次点按钮收起
- [ ] Step 2.8: 浏览器验证：打开任一深色 HTML 封面（如养虾系列），选中文字 → 色板 → 选白色 → 确认文字变白；同样测对齐和链接
- [ ] Step 2.9: CHANGELOG + commit + tag v1.11.1

### 验收标准

- 字色可选白色（可见）
- "最近使用"记忆有效
- 对齐 4 按钮都可用
- 链接可插入、可编辑、可取消

## Task 4 · v1.11.3 · 图片 Notion 式操纵（resize + 段内移动）

**Why:** 用户能替换/删除图片，但不能拖和改尺寸。Notion 是最成熟的 contenteditable 内图片操纵范式。

**Scope:** 核心在 saver-runtime.js，加一个 ImageManipulator IIFE 模块。

**Files:** `saver-runtime.js`

### Steps

- [ ] Step 4.1: 为所有 `<img>` 加事件委托——点击时显示 4 角 resize handle（8×8 蓝色圆点）
- [ ] Step 4.2: 选中态视觉：img 加 outline，四角 handle 绝对定位覆盖在 img 上方
- [ ] Step 4.3: 右下角 handle 拖拽 → 改 `width` + `height`（保持宽高比，按 Shift 时放开）
- [ ] Step 4.4: 拖拽过程中实时更新 img 的 `style.width / style.height`，松手时触发 dirty + 快照
- [ ] Step 4.5: 图片**段内移动**：img 主体拖拽 → 在当前段落内上下移动（不 float、不跨段）
  - 实现思路：检测 parent 是 p/div，拖到 parent 上半/下半就 insertBefore/insertAfter
- [ ] Step 4.6: 取消选中：ESC / 点击外部 / 光标移入别处 → 隐藏 handle
- [ ] Step 4.7: 脏状态集成：resize 和移动都算一次用户操作，走 USER_INTERACT_WINDOW_MS 机制（v1.2.4 护栏不退化）
- [ ] Step 4.8: 保存时不要把 outline/handle DOM 写回文件（serializeHTML 已有清理机制，确认它剔除 `#__dc_toolbar` 那种临时 DOM）
- [ ] Step 4.9: 浏览器验证：多张图片测试 resize + 段内移动
- [ ] Step 4.10: CHANGELOG + commit + tag v1.11.3

### 验收标准

- 点击图片出现 4 个蓝色 handle
- 拖右下角能改尺寸
- 拖主体能在段内移动
- 点外部或按 ESC 取消选中
- 保存后文件里只有 style.width/height 的变化，没有 handle 残留

## Task 5 · v1.11.4 · 元素位置微调（transform translate 方案）

**Why:** 用户想把养虾系列封面 HTML 里"路易乔布斯 © 2026"往上移，当前做不到。这是 AI 生成 HTML 的微调刚需。

**Scope:** saver-runtime.js 新增 ElementPositioner 模块。使用 `transform: translate(X, Y)` 避免破坏布局流。

**Files:** `saver-runtime.js`

### Steps

- [ ] Step 5.1: 设计交互触发：**Alt + 点击任意元素**进入"位置微调"模式
  - 避开和现有 Alt+click 替换图片的冲突（v1.9.2 修过一次）→ 用 **Alt+Shift+click** 或用右键菜单
  - 最终选择：**Ctrl/Cmd + 长按 300ms** 元素 → 进入微调模式（区分度高且零冲突）
- [ ] Step 5.2: 选中元素后在元素旁弹出浮动控制卡片
  - 4 个方向按钮（↑ ↓ ← →）每次按一次移动 1 / 5 / 10 像素（按 Shift 放大档位）
  - X 和 Y 数字输入框可直接敲
  - "重置"按钮清掉 transform
- [ ] Step 5.3: 实时更新元素的 `style.transform = translate(Xpx, Ypx)`
- [ ] Step 5.4: 键盘快捷键：激活卡片时方向键 = 1px，Shift+方向 = 10px，Esc 退出
- [ ] Step 5.5: 保存集成：dirty + snapshot（只有 transform 值变化，diff 很轻量）
- [ ] Step 5.6: 兼容已有 transform：若元素已有 `transform: rotate(...)` 等，解析保留 rotate 部分，只改 translate
- [ ] Step 5.7: 退出模式清理浮动卡片 DOM
- [ ] Step 5.8: 浏览器验证：打开养虾封面，Cmd+长按署名 → 方向键上移 → 保存 → 关闭刷新看持久化
- [ ] Step 5.9: CHANGELOG + commit + tag v1.11.4

### 验收标准

- 能选中元素进入微调模式
- 方向键/按钮/数字输入都能改位置
- 保存后文件里有 `style="transform: translate(Xpx, Ypx)"`
- 重置按钮能清掉 transform
- 不影响其他已有 transform 属性（如 rotate）
