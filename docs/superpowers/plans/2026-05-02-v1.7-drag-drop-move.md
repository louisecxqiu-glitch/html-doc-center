# v1.7.0 — 拖拽移动文件 / 目录

## Goal

侧边栏支持拖拽文件/目录到目标文件夹，快速整理文档结构。

**关键约束（已与用户确认）**：
- 单文件/单目录，每次拖拽弹 confirm
- 允许跨 scan_roots 移动
- 目标冲突 → 询问，拒绝移动（不覆盖、不自动改名）
- 不做撤销机制
- 快照按文件名 stem 过滤后一起移动
- 正在编辑（dirty）的文件禁止拖拽，先落盘
- 目标必须是 enabled 的 scan_root 之内的目录

## Architecture

- 后端新增 `POST /api/move`——校验源和目标均在 scan_roots 内，冲突检测，移动文件 + 匹配的快照
- 前端 HTML5 drag-and-drop：`draggable="true"` + `dragover` 高亮目标 + `drop` 弹 confirm → 调 API
- 拖拽不走自动保存，dirty 文件先阻拦

## Tech Stack

Python 3 · aiohttp · HTML5 DnD API · `shutil.move`

## Tasks

### Task 1 — 后端 /api/move

**Files**: `server.py`

**Steps**:
- [ ] 1.1: `handle_move(request)`：body `{src: str, dst_dir: str}`
- [ ] 1.2: `_resolve_safe(src)` 和 `_resolve_safe(dst_dir)` 校验，任一失败返回 403
- [ ] 1.3: 校验 src 存在、dst_dir 存在且是目录
- [ ] 1.4: 计算 `target = dst_dir / src.name`，若已存在返回 409 `{ok: false, error: "conflict", conflict_path: str}`
- [ ] 1.5: `shutil.move(src, target)`；若 src 是文件，额外处理 _auto-save
- [ ] 1.6: `_move_matching_snapshots(old_file, new_file)`：old 文件的 `{parent}/_auto-save/{stem}-*.{suffix}` 和 `{stem}-pre-overwrite-*.{suffix}` 移动到 `{new_parent}/_auto-save/`（自动创建）
- [ ] 1.7: 清 tree 缓存；返回 `{ok, src, dst, new_abs_path, moved_snapshots: N}`
- [ ] 1.8: 路由注册

## 验收标准

- [ ] 从 A 目录拖一个 HTML 到 B 目录 → confirm → 成功，目录树自动刷新，新位置显示
- [ ] 文件被移动后，它的 _auto-save/ 快照也跟着走（只挪同名 stem 的，无关快照保留原位）
- [ ] 拖到已有同名文件 → toast 冲突提示，文件未动
- [ ] 拖到不在任何 scan_root 下的目录 → 403
- [ ] 正在编辑 dirty 的文件 → 被 preventDefault，提示先保存
- [ ] 拖文件夹到自己的子目录 → 前端拦截，不发请求

## 回退预案

- 代码层：`git revert v1.7.0-commit`
- 文件层：若用户反馈某次移动后找不到文件，可从 git log 里找到那次 /api/move 调用的 src/dst（服务端日志会打印），手动 mv 回去
- /api/move 每次调用都会 `_log()` 打印 src → dst，作为事件日志
