1. AI时代用户的8个真实任务

👓 Christensen · Jobs-to-be-Done — 用户不"买"产品，用户"雇佣"产品来完成任务。

AI时代的内容生产链路：人 → AI工具 → 生成HTML产物 → ??? → 最终交付。问号这段是断裂的——用户有8个任务没人帮他完成：

#任务（用户内心独白）当前怎么解决痛点强度谁在受苦
1"AI生成了一份报告，我要先看看它做得怎么样"双击HTML文件，一个个打开中所有AI工具用户
2"报告里有几个错字和数据要改，不想再让AI重新生成等半天"回AI重新生成（等5分钟），或自己改源码（不会）高非技术用户尤其痛苦
3"AI生成了5个版本，我想对比哪个最好"手动重命名+逐个打开对比，靠脑子记差异高频繁用AI生成的人
4"我要把报告发给老板/客户看"导出PDF发微信，或发HTML对方打不开高所有需要交付的人
5"团队要一起审阅这份AI报告"微信群里截图讨论低团队协作场景
6"我在公司电脑做的，想回家继续改"U盘拷贝/发自己微信中多设备用户
7"我不会编程，不想装Python敲命令行"找技术同事帮忙装，或直接放弃高非技术用户（主流市场）
8"这是公司数据，不想传到陌生服务器上"只能本地操作，不敢用云工具中企业/合规敏感用户

2. 第一性原理：内容生产链路的断裂

⚡ Musk · 第一性原理 — 从基本事实出发重新推导，质疑所有"理所当然"的假设。

传统时代 vs AI时代的内容生产

传统时代：人 → 工具 → 内容（人用工具从零创造内容）

AI时代：人 → AI → 内容 → 人 → 精修 → 最终内容（AI生成初稿，人审阅+精修）

关键变化：内容的"初稿"由AI生成，人的角色从"创造者"变成"审阅者+精修者"。

三个推论

推论1
不需要强大的"从零创作"能力。StackEdit的MD写作路径是上个时代的——AI时代用户不从空白页开始。
推论2
需要"快速查看+轻量修改+版本管理+分享"的完整链路。这是AI产物的生命周期管理。
推论3
AI会反复生成，版本管理是核心需求而非feature。这是HotPage与所有竞品的根本差异——drawio/StackEdit都没有。
洞察：HotPage不是"HTML编辑器"，是"AI产物生命周期管理工具"。编辑只是其中一个环节。定位的转变决定了功能优先级——版本管理和分享比编辑器增强更重要。

4. 功能清单：从用户任务倒推

🧱 Krug · Don't Make Me Think + 🌊 Moore · Crossing the Chasm — 获客层（让用户用起来）→ 留存层（让用户留下来）→ 商业化层（让用户付费）。

第一层：获客层 — 让用户能用起来

解决任务T7（零门槛）+ T4（分享）。没有这一层，非技术用户根本进不来。

#功能解决任务理由
F1单文件可执行版（PyInstaller打包）T7 零门槛当前获客第一瓶颈是"需Python+命令行"。PyInstaller把server.py打包成.app/.exe，双击即用。改动最小（server.py几乎不改），见效最快。drawio的桌面版就是这个逻辑。
F2单文件离线分享（HTML+依赖base64内嵌）T4 分享当前"发HTML对方打不开"是高频痛点。把HTML+CSS+JS+图片全部base64内嵌成一个自包含文件，对方双击即看。HotPage已有base64图片能力，扩展到全资源即可。
F3PWA在线版（公开网址零安装）T7 零门槛 + T4 分享终极获客入口——打开网址即用，无需安装。用File System Access API替代Python后端。这是drawio最大的获客渠道（app.diagrams.net打开即用）。成本高但杠杆最大。

第二层：留存层 — 让用户留下来

解决任务T3（版本对比进阶）+ T2（微调进阶）+ T4（分享进阶）。用户用起来后，这些功能让他不舍得走。

#功能解决任务理由
F4版本对比视图（并排diff+高亮变化）T3 版本对比HotPage已有行级diff数据（difflib），但只在时间线里显示文本diff。加一个视觉对比——左右并排渲染两个版本，高亮变化区域。这是HotPage独占能力的可视化升级，让"AI反复生成"的用户能直观看到差异。
F5AI辅助编辑（选中段落→AI重写/美化/翻译）T2 微调进阶当前微调是手动改字。加"选中段落→右键→AI重写/翻译/润色"——把AI能力从"外部生成"延伸到"内部精修"。这是HotPage从"编辑器"升级为"AI原生工作台"的关键。差异化于drawio/StackEdit（它们没有AI）。
F6在线分享服务（短链分享，接收方无需安装）T4 分享进阶F2是离线分享（发文件），F6是在线分享（发链接）。部署share.hotpage.app/{id}，上传HTML生成短链，对方打开即看。类似Carbon的分享模式。这是从"本地工具"到"有网络效应的产品"的关键一步。

第三层：商业化层 — 让用户付费

解决任务T5（协作）+ T6（跨设备）+ T8（企业隐私）。这些是企业/团队才会付费的。

#功能解决任务理由
F7跨设备同步（可选云同步）T6 跨设备本地优先是HotPage的隐私优势，但跨设备是真实需求。解法：可选开启云同步（用户主动选择），数据加密后同步。免费用户不同步，Pro用户可同步。借鉴StackEdit的多云策略但不强制。
F8团队协作（多人批注+冲突检测）T5 协作当前只有单人划词批注。升级为：多人可加批注+@提及+批注状态（待处理/已解决）。不做实时协作（工程量太大且偏离定位），做异步审阅协作——更贴合"AI报告审阅"场景。
F9企业自托管版T8 企业隐私企业版：自托管部署+SSO+审计日志+权限管理。面向有合规要求的企业。drawio的自托管模式证明这个市场存在。这是营收天花板最高的功能。
F10模板/提示词市场新需求AI生成HTML需要好的提示词/模板。社区贡献模板（如"季度报告模板""数据仪表盘模板"），用户一键使用。这是生态功能，有网络效应。长期可成为第二增长曲线。

6. 商业化路径：免费→Pro→Team→Enterprise

🌊 Moore · Crossing the Chasm — 免费版获客，Pro版留存变现，Team/Enterprise版规模化营收。

Free · 获客层

$0/月

  • 单文件可执行版（桌面）
  • 基础编辑+版本管理（限20个快照）
  • 单文件离线分享
  • 本地文件管理（无限文件）

目标：降低门槛到零，让非技术用户也能用。这是引流入口。

Pro · 留存变现

$9.9/月

  • 无限版本快照+版本对比视图
  • AI辅助编辑（每月500次）
  • 在线分享服务（无限短链）
  • 跨设备云同步
  • PWA在线版

目标：个人用户付费。核心卖点是"AI辅助+版本对比+跨设备"。

Team · 团队变现

$19.9/人/月

  • Pro全部功能
  • 团队协作（多人批注+@提及）
  • 共享工作空间
  • 权限管理
  • 团队模板库

目标：小团队AI报告审阅协作。异步审阅模式，不做实时协作。

Enterprise · 企业

询价

  • Team全部功能
  • 自托管部署
  • SSO+SAML
  • 审计日志+合规报告
  • 专属支持+SLA

目标：有合规要求的企业。营收天花板最高。

商业化关键洞察：免费版必须足够好（能浏览+编辑+版本管理+离线分享），让用户用起来；Pro版用"AI辅助+版本对比+跨设备"让个人用户付费；Team版用"协作审阅"让团队付费。免费→Pro的转化钩子是"版本快照超限"和"AI辅助编辑次数用完"。

8. 立即执行清单

Phase 1 立即执行（2周内完成）
  1. F1: PyInstaller 单文件可执行（1周）
    • 写 build.py 打包脚本（PyInstaller --onefile server.py + web/ 资源）
    • Mac 签名+公证（解决"无法验证开发者"）
    • Windows exe 打包
    • GitHub Release 发布 v2.0，提供 .dmg/.exe 下载
    • README 首页改为"下载安装"而非"clone+pip"
  2. F2: 单文件离线分享（3天）
    • saver-runtime.js 新增"导出自包含HTML"功能
    • 把 HTML 内的 <link> <script> <img> 全部内联/base64
    • 工具栏新增"📦 分享"按钮
    • 导出后自动复制到剪贴板
  3. F4: 版本对比视图（1周）
    • 时间线 UI 新增"对比"按钮
    • 左右并排 iframe 渲染两个版本
    • 高亮变化的 DOM 区域（基于 diff 结果）
    • 支持"当前版本 vs 任意历史版本"
Phase 2 规划（Phase 1 完成后启动）
  1. F5: AI辅助编辑 — 选中段落→右键→AI重写/翻译/润色。接入 Claude API（用户自带 key 或平台代理）。
  2. F6: 在线分享服务 — 部署 share.hotpage.app 后端（存储+短链），前端"分享"按钮上传生成链接。
  3. F3: PWA在线版 — 剥离 app.js 对 Python API 的依赖，改用 File System Access API。部署 hotpage.app。
产品经理的判断：Phase 1 的三件事（F1+F2+F4）是"让产品可交付"的最低要求。做完这三件，HotPage才能从"Louis的个人工具"变成"可以给别人用的产品"。在此之前谈商业化都是空中楼阁。

优先级铁律：先让用户用起来（F1），再让用户能分享（F2），最后让用户看到独特价值（F4）。这个顺序不能颠倒——用户如果装不上，后面再多功能都没用。
