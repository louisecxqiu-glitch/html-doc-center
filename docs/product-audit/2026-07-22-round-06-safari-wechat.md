# 产品审计 · Round 06（Safari 公众号兼容性）

**角色**：在 macOS Safari 中编写、排版并发布公众号文章的作者
**日期**：2026-07-22
**环境**：Safari（本地功能分支服务 `127.0.0.1:9904`）

## 过程与证据

- Safari 成功打开相同的 Markdown 文档，文本、表格、图片和预览均已渲染。
- 点击「公众号」后，初始状态短暂显示「排版中…」，随后变为「已更新」，并加载 `about:srcdoc` 的公众号预览；预览包含标题、列表、表格、图片和链接。
- 以最新页面状态点击「复制 HTML」后，界面显示 **「已复制，可粘贴到公众号」**。

首次点击期间，桌面自动化通道报告 Safari 应用状态变化，未能证明复制动作已执行；重新获取最新 UI 状态并重试后得到成功反馈。因此将这次异常归为测试控制通道的状态失效，而非产品复制失败。

## 结论

- Safari 的公众号预览和富文本复制流程通过。
- 仍未向真实公众号后台粘贴：该操作会把本地文章内容发送给第三方，且不属于本轮必要的本地回归。
- 工作区选择由浏览器自身的本地存储隔离；Safari 首次打开默认显示 All workspaces，而文档顶部仍标示当前文件所属的 `html-doc-center` 工作区。这符合每个浏览器独立保存偏好的预期，但首次使用者仍需要从选择器手动聚焦子工作区。

## 付费判断更新

核心发布链路现已在 Chromium 和 Safari 都获得了真实 UI 成功反馈。我会愿意进行小范围付费试用；在推荐稳定付费前，最后一项关键验收是使用无用户内容的临时样本完成一次「自动快照 → 查看 → 恢复」演练。
