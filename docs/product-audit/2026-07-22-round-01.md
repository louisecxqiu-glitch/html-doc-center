# HTML Studio 产品体验审计 · 第 1 轮

**日期：** 2026-07-22
**角色：** 自媒体内容创作者、需要审阅和保管 HTML 成果的上班族
**版本基线：** v2.7.0（工作区另有未提交的用户改动，未纳入本轮）

## 方法与范围

在本机启动当前代码，通过真实浏览器完成以下路径；未修改源文档、未写入用户内容。

1. 打开 Markdown 教程，进入“公众号排版”并检查排版预览、图片与状态反馈；
2. 验证“复制 HTML”动作及页面状态；
3. 从主界面打开 Markdown，查看文件树、搜索、状态栏和视图切换；
4. 打开历史版本抽屉，检查空状态、自动快照说明和恢复入口。

## 发现

### P0 — 无

未发现数据丢失、越权读写或无法打开核心文档的证据。

### P1 — 公众号操作在错误上下文暴露

**复现：** 打开 Markdown 后保持“原文”视图。
**现象：** “复制 HTML / 导出 HTML”仍显示在工具栏中，虽然尚未进入“公众号排版”。
**影响：** 创作者会误以为当前已经生成了可发布版本；也增加了工具栏噪音。
**证据：** `#md-wechat-actions` 带有 `hidden` 属性，但 `.md-wechat-actions { display: inline-flex; }` 未显式处理 `[hidden]` 状态，真实界面仍显示两个动作。
**建议：** 显式添加 `.md-wechat-actions[hidden] { display: none; }`，并增加真实 DOM 行为测试：仅 `data-view-mode="wechat"` 时可见。

### P1 — 大型多根目录缺乏“当前工作区”边界

**复现：** 当前文件树加载 4 个根目录、7125 个文档，搜索 `macos-dmg-distribution.md`。
**现象：** 结果出现两条同名命中；文件树只能靠较长的路径和展开层级判断来源。
**影响：** 上班族处理客户文件或多项目资料时，难以确认搜索范围与当前正在编辑的文件集合；误开、误改风险高。
**建议：** 在侧栏提供当前根目录范围指示与筛选；搜索结果显示精简、可辨认的根目录标签，并去重相同绝对路径。

### P2 — 启动文档与实际命令不一致

**复现：** 执行 README 推荐的 `python3 server.py --dev`。
**现象：** 命令报“unrecognized arguments: --dev”；实际支持 `--open-browser`、`--no-open-browser`、`--port` 和 `--host`。
**影响：** 新用户按文档启动即失败，损害首次体验。
**建议：** 修正文档，或有意实现 `--dev` 的语义并测试。

## 付费意愿判断

**当前：愿意继续试用，不建议为专业版付费。**

我认可“本地 HTML/Markdown 统一浏览、即时修改、历史保护、公众号输出”的组合价值；但专业用户会先问两个问题：

1. 我现在操作的文件是否属于正确项目、搜索结果是否可靠？
2. 发布动作是否只会在正确的发布模式中出现？

修复 P1 的上下文一致性与 P1 的工作区边界后，再复测编辑、保存和恢复闭环，才能进入“愿意付费”的判断。
