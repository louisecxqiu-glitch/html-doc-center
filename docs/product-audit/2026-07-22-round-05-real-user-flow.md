# 产品审计 · Round 05（真实用户流回归）

**角色**：需要管理多目录素材、把 Markdown 教程排版后发布到公众号的自媒体作者
**日期**：2026-07-22
**环境**：Chromium（本地功能分支服务 `127.0.0.1:9904`）

## 已完成的真实流程

1. 打开 `macos-dmg-distribution.md`，确认顶部显示 `⌂ Workspace: html-doc-center`。
2. 在工作区选择器选择 `html-doc-center`：文件树收敛为 **1 root(s) · 91 document(s)**，避免父目录与子目录同时扫描时的重复内容。
3. 打开「公众号」模式：实际出现苹果风格预览、`复制 HTML` 与 `导出 HTML` 操作。
4. 点击 `复制 HTML`：界面返回 **「已复制，可粘贴到公众号」**，说明 Chromium 的富文本剪贴板主路径完成。
5. 打开历史：抽屉可打开、当前文件名称正确，并明确说明自动快照的触发与恢复保护规则。此次文档未编辑，因此没有伪造快照来验证恢复。
6. 打开设置：嵌套工作区提示可见——All 模式只保留父目录，用户可从顶部工作区选择器聚焦子目录。
7. 刷新页面：仍恢复为 `html-doc-center` 工作区、**1 root(s) · 91 document(s)**，并重新打开当前 Markdown 文件。

## 结论

- **修复后验证通过**：工作区作用域、嵌套目录提示、公众号预览、Chromium 富文本复制主路径、页面刷新后的作用域持久化。
- **不把未验证当作已验证**：本轮没有向真实公众号后台粘贴，也没有在禁用 `ClipboardItem` 的浏览器中触发回退路径；该回退路径由静态契约与全量测试覆盖，仍需在 Safari/受限权限环境手工验收。
- **体验观察**：历史抽屉在没有版本时解释充分，但对第一次使用者而言，「自动快照不会写回源文件」值得在文案中再强调；先放入后续易用性候选项，不阻塞当前功能分支。
