## 本轮实际缺口与修复

- **问题**：不支持 `ClipboardItem` 的浏览器会执行 textarea 复制，剪贴板里是 HTML 源码；用户粘贴到公众号后得到的是代码文本，和“已复制，可粘贴到公众号”的反馈矛盾。
- **根因**：回退逻辑复制的是字符串，而不是浏览器可序列化为 `text/html` 的 DOM 选区。
- **解法**：回退路径创建临时、离屏的 `contenteditable` 容器，选中格式化 HTML 后调用 `execCommand('copy')`；现代剪贴板 API 因权限或 MIME 兼容性失败时也会继续尝试该路径。`finally` 始终清理临时选区和 DOM 节点；两种机制都失败时才提示用户导出 HTML。
