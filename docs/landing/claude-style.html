The files AI generates pile up faster than you can read them. DocCenter is a quiet workbench for browsing, editing, and quietly archiving every version you ever cared about — without Notion, without GitHub, without leaving your machine.
What it does

Three things, done well.

Most tools try to be everything. DocCenter is opinionated: it solves three problems the AI era invented, and stays out of your way for the rest.

— 01 —

Three-view Markdown editing.

Source. Split. Preview. A drag-resizable splitter sits between editor and preview. Your view mode and ratio persist across sessions — close the tab, reopen, find it exactly as you left it.

No external editor required. No plugin to install. Just open .md and start typing. Switch view in one click.

— 02 —

Never lose what you wrote.

Stop typing for two seconds. DocCenter writes a snapshot. Forever. A time machine bar lets you jump back five minutes, an hour, a day. The history drawer shows line-level diffs and restores anything with a pre-restore backup of the current state.

No git commit ceremony. No "wait, did I save?" anxiety. Snapshots are plain files in _auto-save/, gitignorable and inspectable. Gradient thinning prevents the timeline from exploding.

— 03 —

Everything you wrote, in one tree.

Point DocCenter at the folders where your AI-generated files live. It builds a single navigable tree across them — HTML reports, Markdown drafts, deck covers, every artifact. Click to render. Click to edit. The browser becomes your studio.

Three-tab sidebar (Tree / Favorites / Recent). Path search with hit highlighting. Bilingual UI. Dark mode that follows your system. Right-click for context actions. Open any file in a new tab for side-by-side reading.

Why this exists

A note from the author.

I built DocCenter because I was drowning in HTML files.

Every evening I would generate five decks, three reports, two covers with Claude. By the next morning they were lost — scattered across outputs/, drafts/, ~/Desktop/. To fix a typo, I would open VS Code, find the file, edit the source, refresh the browser. Five steps for a typo.

And every once in a while, I would lose the good version — the one with the larger font, the slightly different gradient, the bullet I would never write again.

So I wrote a small Python file. It scans my folders, renders my HTML in a browser, injects a quiet editor, and snapshots everything I type. Fifty-one versions later, it became the tool I open every morning before anything else.

It is local-first because my work is local. It is zero-dependency because dependencies are debt. It is open source because the problem isn't mine alone.

If you also generate too much HTML, this might be for you.

— Louis Qiu, June 2026

Get started

Three commands.

Python 3.8 or newer. No npm, no docker, no account. If you have Python, you have DocCenter in about thirty seconds. 

# Clone the repository
$ git clone https://github.com/louisecxqiu-glitch/html-doc-center.git
$ cd html-doc-center

# Install the single dependency
$ pip3 install aiohttp

# Start the workbench
$ python3 server.py

# Open in your browser
$ open http://localhost:9901

First-time setup creates ~/.codebuddy/html-doc-center/config.json. Add directories you want to scan via the gear icon (⚙) in the sidebar footer. Settings are remembered across restarts.
