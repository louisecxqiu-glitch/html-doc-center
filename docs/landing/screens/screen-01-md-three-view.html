ARIS：对抗式 Multi-Agent 协作

核心问题

长 Agent 跑 8 小时后会出现"看似合理但缺乏证据"的失败模式——这是单 Agent 的根本缺陷。

解法：跨家族 Reviewer

Executor 用 Claude，Reviewer 用 GPT-5。Reviewer 不允许总结，必须直接读源。

三层保障栈

  1. /experiment-audit — 检测 5 类失败
  2. /result-to-claim — 主张账本
  3. /paper-claim-audit — 零上下文交叉核对

关键设计

  • 单 Agent 自审 ≈ 噪声 bandit
  • 双 Agent 跨家族 ≈ 对抗 bandit
  • 更快收敛到 Nash 均衡
